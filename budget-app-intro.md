# 📱 记账软件 — 完整介绍

> **轻量 · 全功能 · 零依赖**  
> 一个 HTML 文件搞定个人记账，开箱即用，无需安装，无需联网。

---

## 📌 概述

**记账软件** 是一款纯前端个人财务追踪应用，采用**单页 HTML 文件**架构，所有代码（HTML + CSS + JavaScript）整合在一个文件中。数据存储在浏览器的 `localStorage` 中，不经过任何服务器，完全离线可用。

| 项目 | 说明 |
|------|------|
| 界面语言 | 全中文 |
| 货币单位 | **马来西亚令吉 (RM)** |
| 适用平台 | 桌面浏览器 / 手机浏览器 / 平板 |
| 技术栈 | 纯 HTML + CSS + JavaScript（零外部依赖） |
| 数据存储 | 浏览器本地存储 (`localStorage`) |
| 文件大小 | ~150KB |

## 📄 六大核心页面

### 1️⃣ 总览页 — Overview

进入应用第一屏，快速掌握本月财务状况。

#### 核心指标卡
| 指标 | 说明 |
|------|------|
| 本月总支出 | 当月所有消费汇总 |
| 月度预算 | 设定的零花钱金额 |
| 日均支出 | 按已过天数计算的平均每日消费 |
| 预测月总 | 按当前消费习惯推算的月末总支出 |

#### 预算进度环
- 环形图直观显示预算使用率（已花 / 可支配金额）
- 可支配金额 = 零花钱 − 储蓄目标
- 超支时自动变为红色，并显示超支金额
- 下方显示计算过程：`零花钱 RM1500 − 储蓄 RM500 = 可支配 RM1000`

#### 储蓄目标环
- 追踪实际储蓄 vs 目标储蓄的进度
- 达成后显示「✅ 已达成」
- 未达成时显示百分比进度

#### 💵 储蓄预测卡
显示四列数据：
- **当前已存**：可支配金额 − 已支出
- **储蓄目标**：设定的目标金额
- **预计月末储蓄**：按当前消费习惯推算
- **目标达成率**：百分比进度条

同时显示提示文字：
- ✅ 如果维持消费习惯，月末预计可存 XX
- ⚠️ 预计超支 XX，请注意控制

#### 其他模块
- **近 7 天趋势图**：迷你折线图显示每日支出波动
- **相比上月**：环比变化百分比（绿色↓ / 红色↑）
- **支出排行 TOP 5**：本月花钱最多的分类排行
- **快捷操作**：「记一笔」和「查看统计」按钮

### 2️⃣ 记一笔页 — Add Record

新增消费记录。

- **金额输入**：自动过滤非数字字符，保留两位小数，移动端弹出数字键盘
- **分类选择**：弹出树形分类弹窗，支持无限级嵌套，展示完整层级路径
- **日期选择**：默认当天，可自行修改
- **备注输入**：可选文字说明

---

### 3️⃣ 流水页 — Records

查看、搜索、管理所有消费记录。

#### 🔍 高级筛选
支持多条件组合筛选：
- 关键词（搜索备注内容）
- 分类（弹出分类树选择）
- 日期范围（起始 + 结束日期）
- 金额范围（最低 + 最高金额）
- 一键清除所有筛选条件

#### 📋 两种视图模式

| 卡片视图（默认） | 紧凑视图 |
|:---:|:---:|
| 大圆图标 + 分类名竖排 | 日期、分类、备注、金额横排一行 |
| 日期、备注换行显示 | 备注自动截断，适配长文本 |
| 触摸滑动可删除 | 🗑️ 删除按钮常驻可见 |
| 适合浏览详情 | 适合快速扫描大量记录 |

两种视图通过按钮一键切换，偏好自动保存到 `localStorage`。

#### 记录操作
- **点击编辑**：点击任意记录打开编辑弹窗，修改金额、分类、日期、备注
- **删除记录**：
  - 桌面端：鼠标移到卡片上即显示 🗑️ 按钮
  - 移动端：向左滑动卡片露出「删除」按钮
  - 编辑弹窗内也提供删除按钮
- **无限滚动**：向下滚动自动加载更多历史记录
- **记录计数**：筛选结果实时显示记录总数

#### 📤 数据导出
- **Excel 导出（含真实公式）** — 一键生成可修改的 `.xlsx` 文件
- **CSV 导出** — 通用表格格式
- **JSON 导出/导入** — 完整数据备份与迁移

- **展开/折叠**：树形结构可逐层展开查看子分类

#### 预设分类

#### 📊 图表总览

| 图表 | 数据维度 | 交互 |

- 点击饼图区块或**右侧图例** → 进入该分类的子分类明细

#### 统计筛选

- **自定义日期范围**：任意起始/结束日期，自动修正颠倒
- **日期切换时自动重置钻取状态**

#### 统计分析卡片
- 总支出、记录数、预测月总、储蓄预测
- 单笔最高消费（含分类）、最高消费日
- 环比变化百分比（相比上月/上一同等时长周期）
- 剩余天数日均预算提示
- 储蓄预估详情（零花钱、已存、预计月末储蓄、日均可支配）

## 📤 Excel 导出详解

生成的 Excel 文件包含 **5 个工作表**，全部使用真实公式：

| 工作表 | 内容 | 公式示例 |
|--------|------|----------|
| **消费记录** | 全部逐笔明细 + 合计统计 | `=SUM()`, `=AVERAGE()`, `=MAX()`, `=MIN()` |
| **分类统计** | 根分类 + 子分类金额与占比 | `=SUM()`, `=IF(..., /, 0)` |
| **月度统计** | 每月汇总：支出、日均、预算、可支配、储蓄、使用率 | `=AVERAGE()`, `=IF(RC[-1]>0, RC[-1]-RC[-5], ...)` |
| **预算跟踪** | 预算 vs 实际：剩余金额、使用率、自动状态判断 | `=IF(RC[-1]>=0, "✅ 正常", "⚠️ 超支")` |
| **储蓄统计** | 每月实际储蓄、目标、达成率 | `=MIN(1, MAX(0, RC[-1]/RC[-4]))` |

**特点**：
- 所有合计、占比、剩余、达成率均为 Excel 公式，修改数据后自动重算
- 表头冻结（滚动时固定）
- 自动筛选已开启
- 兼容 Excel / WPS / LibreOffice / Google Sheets

## 🖱️ 交互细节

### 饼图悬停高亮
当鼠标移动到饼图区块上时：
1. 区块向外**弹出 10px**（沿中轴线方向）
2. 颜色**加深 18%**（更有「按下」实感）
3. 添加**投影阴影**（浮起效果）
4. 添加**白色描边**（高光线框）
5. 光标变为 `pointer`

鼠标移出或离开饼图区域自动恢复。

### 动画效果
- 饼图首次绘制有**生长动画**（600ms ease-out cubic）
- 记录卡片有 hover 微交互（上浮 1px + 阴影加深）
- 底部导航栏切换有平滑过渡

### 日期范围自动修正
当选择的自定义结束日期早于开始日期时，自动交换两者并提示「日期范围已自动修正」。

### 金额输入智能过滤
- 仅允许数字和小数点
- 最多保留两位小数
- 不阻塞键盘输入（使用 `oninput` 而非 `onkeypress`）

## 🔧 技术架构

```
budget-app.html
├── CSS (~600行)
│   ├── 设计系统 (CSS 变量)
│   ├── 排版与网格
│   ├── 组件样式 (按钮/卡片/表单/导航)
│   ├── 记录卡片 & 紧凑视图
│   ├── 动画与过渡
│   └── 响应式适配
├── HTML (~100行)
│   ├── Toast 通知容器
│   ├── Modal 弹窗
│   ├── 侧边栏导航 (桌面)
│   ├── 底部导航 (移动端)
│   └── 6 个页面容器
└── JavaScript (~3400行)
    ├── 工具函数 (uuid/formatMoney/etc.)
    ├── DataStore (CRUD + localStorage 持久化)
    ├── StatsEngine (数据分析引擎)
    ├── exportToExcel (XML Spreadsheet 生成)
    ├── 6 个页面渲染函数
    ├── Canvas 图表绘制 (饼图/折线图/柱状图/环形图)
    ├── 分类树管理
    ├── 记录筛选与分页
    ├── 弹窗与 Toast
    └── 事件绑定与交互
```

### 关键技术决策
- **零外部依赖**：不引入任何第三方库，减少安全风险和加载时间
- **Canvas 图表**：使用原生 Canvas API 绘制所有图表，避免 SVG/DOM 性能开销
- **XML Spreadsheet 2003**：Excel 导出使用该格式，无需 ZIP 库即可生成含公式的 `.xlsx` 文件
- **`setTransform` 替代 `scale`**：避免多次重绘时变换矩阵累加导致的图形偏移
- **CSS 控制画布尺寸**：移除 JS 中对 `canvas.style.width` 的覆盖，避免布局反馈循环

## 🚀 快速开始

1. 在浏览器中打开 `budget-app.html`
2. 进入**设置页**设定月度零花钱和储蓄目标
3. 去**记一笔**开始记录支出
4. 定期查看**总览页**和**统计页**了解财务状况
5. 需要存档时导出 Excel 或 JSON

## 📋 使用建议

- **设定预算**：建议先设定月度零花钱，让预算环和储蓄预测生效
- **建立分类体系**：根据个人消费习惯建立合理的分类树（如「餐饮 → 午饭/晚饭/零食」）
- **养成记录习惯**：每次消费后立即记录，避免遗忘
- **定期复盘**：每周查看统计页面，分析消费趋势
- **导出备份**：建议每月导出一次 JSON 或 Excel 作为数据备份
