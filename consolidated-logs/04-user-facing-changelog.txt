### Build 1 — 2026-07-09 20:43 UTC: Initial Release

**Summary:**
A complete single-page personal budget tracking application was built and polished. The entire app is contained in one HTML file (104 KB, 2,895 lines) with zero external dependencies. Open it in any modern browser — no installation or internet required.

**Features included:**
- Dashboard (总览): Monthly spending overview with 4 summary cards, budget and savings progress rings (Canvas), 7-day spending trend sparkline, Top 5 spending categories, overspend warnings, savings prediction with daily remaining limit
- Add Record (记一笔): Amount input with RM prefix, category selector with tree picker (emoji + color), date/time picker, notes field, validation and success toast
- Records (流水): Card list sorted newest-first, tap card to edit, swipe to delete, advanced search (keyword, date range, category filter, amount range), results count and infinite scroll
- Categories (分类): 8 root categories with 21 sub-categories pre-loaded, recursive tree with expand/collapse, add root/sub categories, rename, change emoji icon, change color, move with cycle prevention, delete with cascade options
- Statistics (统计): Month selector or custom date range, summary cards, interactive pie chart (click to drill down), smooth-curve line chart with gradient fill, color-coded bar chart, all charts have "Download PNG" buttons
- Settings (设置): Monthly budget, savings target (fixed/percentage/both), JSON export/import (replace or merge), CSV export, clear all data

**Bug fixes applied:**
- Fixed `selectedCategoryId` state leak between pages (added null resets in `renderAddPage()` and `closeModal()`)
- Fixed `selectedCategoryId` not resetting after edit record submission
- Fixed inefficient pie chart click handler reassignment (added `canvas._pieClickSetup` guard flag)
- Fixed redundant code in `confirmDeleteCategory` (merged duplicate branches)
- Fixed missing `updatedAt` timestamp on record updates

**UI polish applied:**
- Enhanced page transitions with scale effect
- Smooth spring-like toast animations
- Frosted-glass modal backdrop (`backdrop-filter: blur(4px)`)
- Spring overshoot modal scale animation
- Tactile button press feedback (`::after` pseudo-element)
- Card hover transitions with border and shadow effects
- Improved bottom navigation touch targets
- Enhanced responsive breakpoints for mobile/tablet/desktop

**Agents involved:**
- organizer: Initial HTML + CSS framework
- writer: Data layer, pages, charts, statistics engine
- ui-agent: Visual polish, animations, responsive refinement
- tester: Syntax verification, bug detection, regression testing
- team-leader: Coordination, bug fixes, logging

**Technical details:**
- Single .html file (104 KB, 2,895 lines)
- Zero dependencies — no CDN, no libraries, no frameworks
- Data stored in browser's localStorage
- Responsive: bottom tab nav (mobile) and sidebar (desktop)
- Hash-based routing (#/overview, #/add, etc.)

**Known limitations:**
- User-provided content (notes, category names) rendered via innerHTML without sanitization (minimal self-XSS risk in local-only app)
- Only pie and line charts have load animations; bar chart does not animate
- Click-based drill-down implemented, but hover tooltips not present
- `updatedAt` field only set on updates, not on creation
- Requires modern browser (ES2020+); Internet Explorer not supported
- Data in localStorage — clearing browser data loses records (use JSON export for backup)

### Build 2 — 2026-07-09 22:46 UTC: Security Fixes

**Summary:**
Three security improvements were applied to the budget application. No changes to appearance, functionality, or behavior.

**Fix 1 — Content Security Policy (CSP):**
A `<meta>` tag was added that tells the browser to block all external resources and only allow inline styles and scripts. This prevents malicious scripts from loading from external sources.

**Fix 2 — HTML Injection Protection:**
A sanitizer function (`escHtml`) was added and applied to all 29 places where user-supplied text is inserted into the page's HTML. This prevents attackers from injecting malicious HTML tags through text fields.

Protected fields:
- Notes (both compact view and card view)
- Category names and icons (throughout the UI — pickers, record displays, management dialogs)
- Search keywords (search filter input)
- Category rename/icon/color/move/delete dialogs

**Fix 3 — File Permissions:**
File permissions tightened to `600` (owner read/write only), preventing other users on the system from reading or modifying the file.

**Not changed:**
- Application appearance, functionality, or behavior
- Data storage or export logic
- Canvas charts (they use safe rendering APIs)
- CSV/XML export functions (they use their own quoting)

**Agents involved:**
- team-leader: Task decomposition, all edits, verification, logging

**Notable findings:**
- File already contained `esc()` function for XML export sanitization
- Total of 29 template literal expressions across 25+ locations wrapped with `escHtml()`

### Build 3 — 2026-07-10: Category Accordion Refactoring

**Summary:**
The Categories page was refactored from a cluttered single-row card layout to a clean accordion (手风琴) design.

**Visual Change:**
- Before: Each category was a horizontal card with inline buttons for every operation, with indent-based hierarchy
- After: Clean accordion design with header row (always visible) and expandable body section

**Key Design Decisions:**
- All categories start collapsed (no auto-expand)
- Only click the arrow icon to expand/collapse (not the whole header)
- Only 2 buttons exposed on headers: "添加子分类" and "编辑分类"
- All other operations (rename, icon, color, move, merge, delete) are in the edit modal
- Children use same button treatment as parents
- Budget input always visible on the header row (collapsed AND expanded)
- Deep nesting breadcrumb shown at depth >= 4

**What was removed:**
- Old category tree CSS classes (`.category-tree`, `.category-node`, `.cat-toggle`, `.category-children`)
- 6 individual action buttons per category consolidated into edit modal
- Auto-expand of root categories on page load

**What was added:**
- New accordion CSS (31 selectors)
- New `buildCategoryTreeHTML` with accordion structure
- `toggleCatItem` — expand/collapse with CSS animation
- `buildBreadcrumb` and `expandAndScrollTo` — deep nesting breadcrumb with smooth scroll
- `editCategory`, `selectEditIcon`, `selectEditColor`, `saveCategoryEdit` — consolidated edit modal

**Agents involved:**
- team-leader: All code edits (CSS, JS functions, text updates)
- tester: JS syntax verification (no errors)

**Statistics:**

- CSS selectors removed: 7, added: 31

### Build 4 — 2026-07-10 05:01 UTC: Budget System Enhancement

**Summary:**
Nine changes were applied to enhance the budget system, introducing support for child category budgets, percentage budgets, and an updated selection modal.

**What changed:**

1. **Data Storage:** Budget values now stored as objects `{ value: number, type: 'fixed'|'percent' }` instead of plain numbers, allowing distinction between fixed (RM) and percentage budgets.

2. **Budget Input UI:** The budget input in the category tree now includes a clickable RM / % toggle next to the number field. Clicking it switches the budget type between fixed amount and percentage.

3. **Budget Progress Cards:** Fixed budget reading logic to handle the new object format. Added automatic conversion: percentage budgets are converted to monetary values based on the total monthly budget.

4. **Pie Chart & Bar Chart:** Updated to extract the numeric value from the new budget object format.

5. **Budget Selection Modal:** The "choose which categories to monitor" dialog now shows child categories that have budgets (indented with `->`), using the new object format.

6. **Child Budget Inputs:** Leaf child categories now have their own budget input row below the card, with the same RM/% toggle functionality.

7. **CSS:** Added a transition rule for budget progress bar fills.

**Agents involved:**
- team-leader: Decomposed tasks, executed all edits, verified results

**Known limitations:**
- Two older `getCategoryBudget` call sites (overview template top spending categories, monthly report category table) still treat return value as plain number. These were outside the scope of this update.

### Build 5 — 2026-07-10: Animation & Edit Modal Fixes

**Summary:**
Four improvements were made to the category management interface.

**Fix 1 — Smoother Accordion Animation:**
The expand/collapse animation was choppy because it used `setTimeout` with a 3000px height fallback. Replaced with `requestAnimationFrame` double-buffering — height is computed from actual `scrollHeight`, eliminating the visible "jump".

**Fix 2 — Edit Access for Leaf Categories:**
Leaf categories (no children) had no arrow, making their edit/delete buttons completely inaccessible. Leaf categories now show a gear icon next to the budget input that opens the edit modal directly.

**Fix 3 — Faster Edit Modal:**
The edit modal was slow because it rendered 90+ emoji grid items and 14 color swatches inline every time. The emoji grid and color picker are now shown as simple previews with "更改图标" and "更改颜色" buttons that open the existing (already lightweight) icon/color selection modals.

**Fix 4 — Better Guide Text:**
The categories page now shows category count, accordion usage instructions, and "展开全部" / "折叠全部" links.

### Build 7 — 2026-07-10 01:32 UTC: Pie Chart Drill-Down Fix

**Summary:**
Two issues were fixed related to pie chart expanded view behavior and modal sizing.

**Issue 1 — Pie chart click drill-down in expanded view:**
Problem: Clicking a pie slice in the expanded (modal) view to drill into subcategories didn't work reliably. The old code created a new canvas and tried to wrap the click handler after a fixed 700ms timeout, creating a fragile timing race.

Fix: Instead of creating a new canvas, the code now **moves the original pie chart canvas** into the modal. The existing click handlers are preserved. A new `window._expandedPieActive` flag auto-closes the modal when a slice is clicked in expanded mode.

Three changes:
- `drawPieChart()` click handler — added checks for `window._expandedPieActive`
- `expandPie()` function — rewritten to move original canvas into modal
- `shrinkChart()` function — updated to move canvas back and clear flag

**Issue 2 — Enlarge the modal:**
Problem: Expanded chart modal was limited to 560px wide, feeling cramped on larger screens.

Fix:
- Modal inner container widened from 560px to 720px
- Canvas max-height increased from 55vh to 65vh
- Heatmap grid and day cells enlarged to fit bigger modal

**Verification:**
- JavaScript syntax passes validation
- All old `expandPieChart` references removed
- New `_expandedPieActive` flag properly set, checked, and cleared
- New `_pieOriginalParent` variable properly tracked and cleaned up

## Known Limitations (Consolidated)

From all builds, the following limitations are documented:

1. **HTML Sanitization:** Security fixes applied but the local-only nature means self-XSS risk is minimal.
2. **Bar chart animation:** Only pie and line charts have load animations; bar chart does not animate.
3. **Chart tooltips:** Click-based drill-down is implemented, but hover tooltips are not present in charts.
4. **`updatedAt` on creation:** The `updatedAt` field is only set on updates, not on record creation.
5. **Browser support:** Requires modern browser (ES2020+); Internet Explorer not supported.
6. **localStorage dependency:** Clearing browser data will lose all records. Use JSON export for backup.
7. **Unchanged budget call sites:** Two `getCategoryBudget` call sites (overview template and monthly report table) still treat return value as plain number (need follow-up).
8. **Modal chaining edge case:** If user clicks "更改图标" then immediately cancels, the edit modal state is already closed (acceptable since user can re-open).
9. **Very tall categories:** The rAF-based accordion doesn't set a 3000px fallback, so if a category is extremely tall, animation may truncate (content is static, so scrollHeight is accurate).
