# 个人预算管理 — 功能全景文档

## Personal Budget Management — Full Feature Document

**应用简介：** 这是一个纯前端的个人预算追踪应用，界面为中文，货币单位为马来西亚林吉特（RM）。整个应用以单个 HTML 文件（约 6,759 行）的形式交付，无需任何外部依赖，所有数据通过浏览器 localStorage 持久化存储。应用采用响应式布局，支持桌面侧边栏导航与手机底部标签导航双模式，并包含基于 Canvas 的自绘图表系统。

**快速概览：** 7 个标签页（总览 / 记账 / 流水 / 分类 / 统计 / 月度报告 / 设置），约 6,759 行代码（CSS ~1,070 行、HTML ~60 行、JavaScript ~5,630 行），14 种预设颜色，8 个默认根分类及 22 个子分类，7 种以上的图表类型，5 工作表 Excel 导出。

---

## 1. 核心功能分页详解

### 1.1 总览 (Overview)

**页面标识：** `#page-overview`

**空状态引导：** 当没有任何消费记录时，页面中央显示一张带虚线边框和渐变背景的欢迎卡片，包含应用名称、引导文案以及"记第一笔账"按钮（点击跳转到记账页）。

**四格摘要卡片（网格布局）：**

- **本月总支出** — 当月所有记录的金额总和，主题色显示
- **预算** — 当月设定的月度预算，未设置时显示"未设置"
- **日均支出** — 当月总支出 / 已过天数
- **预测月总支出** — 日均支出 x 当月总天数

**预算进度环：** Canvas 绘制的圆环图（donut chart），显示可支配预算的使用百分比。绿色/蓝色表示正常，红色表示超支。中心文本显示百分比数字。下方显示"已用金额 / 可支配金额"以及零花钱计算明细行（预算 - 储蓄 = 可支配）。

**储蓄目标环：** Canvas 圆环图，显示当前已储蓄金额占储蓄目标的百分比。已达目标时为绿色，部分完成时为琥珀色，未设置时为灰色。

**储蓄预测卡片：** 一个详细的储蓄分析卡片，包含 5 列数据：

- 当前已存
- 储蓄目标
- 预计月末储蓄
- 目标达成百分比
- 预算结余

卡片底部显示智能叙事文案，根据预测结果给出积极或警告消息。在预算已设置的前提下，还显示两个每日限额指示器：

- **总额日均（未扣储蓄）：** 预算 ÷ 剩余天数
- **可用日均（已扣储蓄）：** 可支配预算 ÷ 剩余天数

未设置预算时显示引导提示。

**最近 7 日走势：** 小型 Canvas 趋势线（sparkline），包含线条、渐变填充区域和圆点标记。

**支出排行 Top 5：** 按金额降序排列的前 5 个根分类，每项包含编号徽章（圆形彩色背景数字）、分类 emoji 图标、分类名称、预算状态徽章（超支 / 预算内）、金额（RM）及占比百分比。

**预算进度卡：** 可复用的预算进度卡片组件（详见第 4 节 预算系统）。

**超支警告：** 红色左边框卡片，列出所有超出月度预算 80% 的分类，显示分类图标、名称、消费金额和超支百分比。

**今日/昨日消费卡片（双列网格）：** 显示当天和前一天的总消费金额。昨日的金额以次要色显示，今日的金额以警告色突出（大于 0）或无消费时显示"暂无"。数据基于记录中的日期字段实时计算。

**快捷操作：** 两个大按钮 — "记一笔"（跳转记账页）、"查看统计"（跳转统计页）。

### 1.2 记账 (Add Record)

**页面标识：** `#page-add`

**金额输入：** 文本输入框，左侧固定显示"RM"前缀，支持小数点后两位的实时格式校验。只允许数字和小数点，自动过滤非法字符，限制小数位数为 2 位。

**分类选择：** 按钮形式，点击打开统一树形分类选择器（Modal）。选中的分类在按钮中显示颜色圆点 + emoji 图标 + 分类名称。

**日期 / 时间：** `datetime-local` 输入框，默认填充当前本地时间（含时区偏移计算）。

**备注：** 文本输入框，最大 200 字符，占位文案"例：椰浆饭"。

**提交：** 表单提交时验证金额大于 0 且已选择分类。任一条件不满足时触发抖动动画（shake）并在 Toast 中提示错误。提交成功后将记录写入 DataStore、显示成功 Toast 并重置表单。

**分类选择器：** 复用 `openCategoryPicker()` / `buildCategoryTreePicker()` 组件，以树形缩进展示所有分类（含子分类），点击选择后自动关闭 Modal 并更新显示。

### 1.3 流水 (Records)

**页面标识：** `#page-records`

#### 搜索与筛选栏

- **关键词搜索：** 文本输入框，对记录的备注字段进行不区分大小写的模糊搜索，实时触发过滤
- **分类筛选：** 按钮点击打开树形 Modal，包含"全部分类"选项。选择任意分类后，按该分类及其所有后代分类的 ID 进行过滤
- **日期范围：** 两个 `date` 输入框（开始日期 / 结束日期）
- **金额范围：** 两个 `number` 输入框（最低金额 / 最高金额）
- **仅看超支：** 切换按钮，激活后只显示来自超支分类的记录。通过计算当月各类别消费与预算比较来确定超支分类集合

筛选栏底部包含一排操作按钮：

- **清除筛选** — 重置所有筛选条件
- **超支分类** — 切换超支过滤
- **选择** — 进入 / 退出批量模式
- **导出 Excel** — 直接导出 XML Spreadsheet

#### 显示控制

- **紧凑 / 卡片视图切换：** 按钮切换两种记录展示模式，偏好存储在 localStorage
- **批量模式：** 进入多选模式，每条记录前显示复选框
- **每页条数：** 可配置输入框，范围 5-200 条，偏好存储在 localStorage
- **记录计数：** 实时显示当前过滤后的记录总数，例如"128 条记录"

#### 记录列表与分页

分页控件包含：第一页（«）、上一页（‹）、页码按钮（最多同时显示 7 个）、下一页（›）、最后一页（»），以及"当前页 / 总页数 (总数)"信息。

**紧凑视图：** 单行显示 — 日期 + 分类图标和名称 + 备注 + 金额 + 删除按钮

**卡片视图：** 完整详情 — 分类彩色圆形头像 + 分类名称 + 日期时间 + 备注 + 金额 + 悬停显示的删除按钮

**单条删除：** Modal 确认对话框，确认后执行软删除（记录立即从列表移除但在 5 秒内可撤销），同时显示含"撤销"按钮的 Toast。5 秒后永久清除。若同一记录在待删除期间再次删除则重启定时器。

**编辑：** 点击记录打开 Modal，预填所有字段（金额、分类、日期时间、备注），支持修改后保存或在此 Modal 中直接删除。

**撤销删除：** Toast 中的"撤销"按钮调用 `DataStore.undoDelete()` 将记录恢复到列表头部。若已超时则提示"无法撤销"。

#### 批量操作栏

在批量模式下固定显示在屏幕底部：

- **批量删除：** 确认对话框后对所有选中记录执行软删除
- **批量修改分类：** 打开树形分类选择器，将选中记录统一修改为目标分类
- **取消：** 清空选择、退出批量模式

### 1.4 分类 (Categories)

**页面标识：** `#page-categories`

**树形结构：** 以可展开 / 折叠的树形展示所有分类。根节点默认自动展开。页面顶部显示分类总数。底部有"添加根分类"大按钮。

**每分类操作按钮：**

| 操作 | 按钮 | 功能说明 |
|------|------|----------|
| 展开/折叠 | 切换 | 展开或收起子分类列表 |
| 添加子分类 | 添加 | 弹出 Modal 输入名称和图标（emoji），添加到当前分类下 |
| 重命名 | 编辑 | Modal 中编辑名称 |
| 更换图标 | 图标 | 打开 emoji 网格选择器（约 100 个 emoji 的滚动网格） |
| 更换颜色 | 颜色 | 显示 14 种预设颜色圆形选择器 + 自定义 HEX 输入框 |
| 移动 | 移动 | 打开树形选择器选择新父分类，或移至根级别（含循环引用检测） |
| 合并 | 合并 | 将当前分类合并到其他分类，所有记录重新归类。当有子分类时可选：将子分类移至目标分类下 / 删除所有子分类 |
| 删除 | 删除 | 无子分类时直接确认删除；有子分类时提供两个选项：同时删除所有子分类 / 将子分类移至上级 |

**分类预算输入：** 每行内联显示预算输入框（数字输入）和单位切换（RM / %）。支持两种预算类型：

- **固定金额（RM）：** 输入具体数值
- **百分比（%）：** 输入占月度总预算的百分比，系统自动换算为具体金额

**父子预算校验：** 保存子分类预算时，自动验证子项预算总和不超过父级预算。若超限则弹出错误 Toast 并拒绝保存。

### 1.5 统计 (Statistics)

**页面标识：** `#page-stats`

**日期选择：** 支持两种模式：

- **月份选择器（`<input type="month">`）：** 查看指定月份的统计数据
- **自定义日期范围：** 两个 `date` 输入框，自动检测并交换不合理的起止顺序

两种模式互斥，选择月份时清除自定义范围，反之亦然。

**汇总卡片（Row 1）：** 四格网格：

- **本月总支出**（或自定义范围的"范围总支出"）
- **日均支出**（自定义范围时显示"记录数"）
- **预测月总支出**
- **储蓄预测**（颜色编码：绿色为正、红色为负）

**交易分析卡片（Row 2）：** 四格网格：

- **交易笔数** + 有消费天数
- **笔均金额**
- **最大单笔** + 所属分类
- **最高消费日** + 日期

**储蓄估算卡片（仅月份模式）：** 与总览页类似的储蓄预测卡片，包含预算、当前已存、预计月末储蓄、预算结余，以及双重每日限额（未扣储蓄 / 已扣储蓄）。

**图表区（全部基于 Canvas，全部可下载 PNG）：**

- **日历热力图：** HTML 网格（非 Canvas），基于每日消费与日均可支配目标的比率进行 6 色渐变着色（蓝 → 青 → 绿 → 黄 → 橙 → 红）。包含星期表头、图例、未来日期半透明不可点击。点击任意日期弹出当日记录详情。可展开为独立 Modal 大图。
- **饼图：** 弹性弹簧动画（requestAnimationFrame + ease-out bounce），hover 弹出效果（8px 位移 + 阴影 + 白色边框），点击带有子分类的扇形区进行下钻。标签放置在扇形外围，带半透明背景以便阅读。右侧图例包含预算进度微条。支持触屏事件。
- **折线图：** 800ms ease-out cubic 贝塞尔动画，渐变填充区域，储蓄日均目标虚线参考线，圆点标记，网格和轴标签。
- **月环比对比柱状图（可切换）：** 分组柱状图，当前月（蓝色）与上月（绿色）并排显示，每根柱子上方带数值标签。
- **月度趋势折线图：** 近 6 个月的月度支出折线图，带渐变填充、圆点标记和数值标签。
- **月度储蓄柱状图：** 近 6 个月的储蓄柱状图（绿色为正、红色为负），含零线。
- **预算进度卡：** 同总览页的预算进度卡片组件，支持实心/分段切换、排序和监控选择。

**交互功能：**

- **饼图下钻：** 点击带有子分类的扇形区 → 下钻显示子分类分布，图表上方显示面包屑导航和"返回上级"按钮。图例和标签同步更新。
- **柱状图下钻：** 点击带有子分类的柱状条 → 下钻到子分类层级。
- **饼图展开：** 点击"展开"按钮在 Modal 中显示大饼图，右侧附带分类明细表格（可展开子分类行）。
- **热力图展开：** 点击"展开"按钮在 Modal 中显示大热力图，左侧日期、右侧详情面板。
- **图表 PNG 下载：** 每个图表下方有"下载 PNG"按钮，调用 `canvas.toDataURL()` 触发下载。

### 1.6 月度报告 (Report)

**页面标识：** `#page-report`

**打印优化布局：** 月份选择器 + "打印报告"按钮。点击打印触发 `window.print()`，打印样式通过 `@media print` 实现：

- 隐藏导航栏、按钮、Toast、Modal 等非报告元素
- 卡片无阴影、白色背景、`break-inside: avoid` 防止跨页断裂
- Canvas 自适应宽度

**摘要卡片（四格网格）：**

- **预算**（零花钱）
- **总支出**（颜色编码：未超可支配为绿色，否则红色）
- **储蓄**（正数为绿色，负数为红色）
- **储蓄率**（百分比，绿色 >= 20%，琥珀色 > 0%，红色为负）

**环形图（双环）：** 预算进度环 + 储蓄目标环，Canvas 绘制，居中显示百分比。

**每日趋势走势图：** 小型 Canvas sparkline，展示当月每日消费走势。

**分类明细表：** HTML 表格，列含：

- 分类（图标 + 名称）
- 预算（RM，无预算时显示"—"）
- 支出（RM）
- 占比（%）
- 剩余（RM，颜色编码）
- 状态（预算内 / 超支 / —）

表尾合计行（预算总计 / 支出总计 / 100% / 余额 / 总体状态）。

**支出排行 Top 5：** 编号圆形徽章 + 图标 + 名称 + 金额。

**储蓄预测摘要：** 叙事性文案卡片，左边框颜色编码（绿色表示正向预测、红色表示超支），内容包含：

> "预算 RM X，储蓄目标 RM Y，可支配 RM Z，已消费 RM A，预测月底结余/超支 RM B"

### 1.7 设置 (Settings)

**页面标识：** `#page-settings`

**主题切换：** 深色 / 浅色模式切换按钮，偏好存储在 localStorage。切换时通过 `data-theme` 属性控制 CSS 变量。

**月度预算设置：**

- 月份选择器（`<input type="month">`）
- 预算金额输入框
- "保存预算"按钮

**储蓄目标设置：** 三种模式单选按钮：

- **固定金额：** 输入具体 RM 数值
- **百分比：** 输入占月度预算的百分比
- **两者：** 同时设置固定金额和百分比，目标金额为两者之和

类型切换时动态显示 / 隐藏对应的输入框。

**数据管理卡片：** 包含以下功能按钮：

- **导出 JSON：** 下载完整的 JSON 数据备份文件，命名为 `budget-data-YYYY-MM-DD.json`
- **导入 JSON：** 文件选择器读取 `.json` 文件 → 弹出 Modal 选择导入方式：
  - **替换当前数据：** 完全覆盖
  - **合并到当前数据：** 合并记录、分类、预算等
- **导出 Excel（可修改含公式）：** 下载 XML Spreadsheet 2003 格式文件，包含 5 个工作表和实时公式（详见第 6 节）
- **导出 CSV：** UTF-8 BOM CSV 文件
- **清除所有数据：** 二次确认 Modal 后重置为默认状态

## 2. 图表系统 (Charting System)

所有图表均使用原生 `CanvasRenderingContext2D` 绘制，不依赖任何第三方库。

### 2.1 饼图 (Pie Chart)

- **弹性弹簧动画：** 使用 `requestAnimationFrame`，动画持续 600ms，采用弹性缓出函数（1 - pow(2, -7x) * cos(x * PI * 2 * 1.2)），饼图半径从 30% 弹入到 100%
- **Hover 弹出效果：** 鼠标悬停在可下钻的扇形区时，扇形向外偏移 8px 并叠加阴影和白色边框
- **点击下钻：** 点击有子分类的扇形区推入钻取栈，重绘图表显示子分类分布
- **面包屑导航：** 下钻后显示"← 返回上级"按钮和当前层级路径
- **标签系统：** 扇形区外围放射状排列标签（名称 + 百分比），带半透明白色背景以提高可读性，可下钻的分类标签下方有下划线
- **图例：** 右侧图例包含颜色方块、分类名称、消费金额
- **触屏支持：** 通过 `mousemove` + `click` 事件绑定，兼容触屏设备
- **无数据状态：** 居中显示"暂无数据"

### 2.2 折线图 (Line Chart)

- **动画：** 800ms ease-out cubic 缓出动画（1 - (1-t)^3），线条从左到右逐段绘制
- **渐变填充：** 线条下方区域使用 `createLinearGradient` 从半透明主题色渐变到完全透明
- **储蓄目标线：** 绿色虚线表示日均可支配目标，带数值标签
- **圆点标记：** 每个数据点绘制白色描边的实心圆点
- **网格：** 4 条水平网格线 + 左侧 Y 轴标签
- **X 轴标签：** 根据数据密度自动间隔显示日期

### 2.3 柱状图 (Bar Chart)

- **圆角顶部：** 每个柱状条使用 `roundRect` 绘制圆角（仅顶部圆角），颜色使用分类自身颜色
- **数值标签：** 柱状条上方显示金额
- **名称标签：** 柱状条下方显示分类名称（超过 4 字截断加".."）
- **预算进度微条：** 名称标签下方显示 4px 高的预算使用进度条（绿 / 黄 / 红三色）
- **点击下钻：** 点击有子分类的柱状条推入钻取栈

### 2.4 月环比对比图 (Grouped Bar Chart)

- **分组柱状：** 每个分类两根柱子并排（当前月蓝色、上月上月绿色）
- **数值标签：** 每根柱子上方显示金额
- **图例标识：** 左上角显示"蓝:本月 / 绿:上月"

### 2.5 日历热力图 (Calendar Heatmap)

- **HTML 网格：** 使用 CSS Grid，7 列布局对应星期日至星期六
- **6 色梯度：** 基于消费与日均可支配目标的比率：

| 比率 | 颜色 |
|------|------|
| 0（无消费） | 浅灰（特殊样式） |
| 0.0 - 0.4 | 深蓝 |
| 0.4 - 0.7 | 青色 |
| 0.7 - 1.0 | 绿色 |
| 1.0 | 暖黄 |
| 1.0 - 1.5 | 橙色 |
| 1.5 - 2.0+ | 深红 |

- **未来日期：** 半透明不可点击
- **悬停提示：** 显示具体日期和消费金额
- **图例：** 从"少"到"多"的色阶条
- **点击交互：** 点击日期显示当日记录列表，可编辑或删除
- **展开视图：** 支持在 Modal 中放大查看

### 2.6 环形图 (Donut Chart / Ring)

- **Canvas 绘制：** 使用 `arc` 绘制圆环，`lineCap: 'round'` 使端点圆润
- **动画：** `requestAnimationFrame` 缓出动画
- **中心文本：** 百分比数字
- **超支模式：** 超过 100% 时红色显示超支百分比
- **三种状态色：** 预算环根据使用率变色，储蓄环根据达成状态变色（绿 / 琥珀 / 灰）

### 2.7 走势图 (Sparkline)

- **紧凑 Canvas 线条：** 无轴标签，仅线条 + 渐变填充 + 圆点
- **自动缩放：** 基于数据最大值自适应 Y 轴
- **用于总览页和报告页**

### 2.8 月度趋势折线图

- 6 个月的月度支出折线
- 渐变填充 + 圆点 + 上方数值标签
- X 轴标签格式为"X月"

### 2.9 月度储蓄柱状图

- 6 个月的储蓄柱状
- 正储蓄为绿色，负储蓄为红色
- 虚线零线
- 柱顶 / 柱底显示数值标签

### 2.10 通用图表特性

- **深色模式自适应：** 所有图表通过 `getThemeColors()` 读取 CSS 变量（`--text-primary`, `--text-secondary`, `--text-muted`, `--border`, `--bg`, `--card-bg`），深色模式下自动切换颜色
- **DPR 感知：** 所有 Canvas 根据 `window.devicePixelRatio` 设置内部分辨率以确保高清显示
- **CSS 响应式宽度：** Canvas 通过 CSS `width: 100%` 自适应容器宽度，内部仅设置高度
- **PNG 下载：** `downloadChart(canvasId)` 函数将 Canvas 内容导出为 PNG 并触发下载
- **`roundRect` 兼容：** 为不支持 `CanvasRenderingContext2D.prototype.roundRect` 的浏览器提供 polyfill

## 3. 预算系统 (Budget System)

### 3.1 月度预算

在设置页中为指定月份设定整体预算金额（RM）。未设置时相关功能显示"未设置"。

### 3.2 分类预算

在分类页的树形行内直接设置每个分类的预算，支持两种类型：

- **固定金额（RM）：** 直接输入数值
- **百分比（%）：** 输入占月度总预算的百分比，系统自动换算为具体金额

行内通过点击"RM"或"%"标签进行单位切换。

### 3.3 子分类预算校验

保存子分类预算时自动验证：所有同级子分类的预算之和不得超过父级预算。超限时弹出错误 Toast 并阻止保存。

### 3.4 预算进度卡 (Budget Progress Card)

这是一个在总览页和统计页复用的核心组件，包含以下功能：

**视图切换：** 滑动开关两种模式：

- **实心模式（Solid / 单色）：** 每个分类显示一根单色进度条，颜色根据使用率变化（绿 < 80%、黄 80-100%、红 > 100%）
- **分段模式（Segmented / 分段）：** 父分类的进度条按子分类消费比例分割为多个色段，每个色段可悬停查看具体数值

**排序选项：** 下拉菜单三种排序方式：

- **按使用率：** 预算消耗百分比从高到低
- **按金额：** 消费金额从高到低
- **按名称：** 分类名称字母顺序

**监控选择：** 点击"选择"按钮打开 Modal，通过复选框选择要监控的分类，支持全选 / 取消全选 / 反选。选择结果持久化到 localStorage。当所有带预算的分类都被选中时，存储为 null（表示全部显示）。

**显示内容：** 每行包含分类图标 + 名称 + 已消费金额 / 预算金额 + 百分比 + 超支徽章 + 进度条。子分类预算行（设有独立预算的子分类）缩进显示在父分类下方，并标注所属父分类。子分类进度条为实心单色条，父分类支持实心 / 分段（按子分类消费比例分割）两种模式。子分类行跟随父分类排序，不独立参与全局排序。

**总计行：** 底部灰色分割线上方显示汇总数据（合计消费 / 合计预算 / 总使用率）。

## 4. 储蓄系统 (Savings System)

### 4.1 储蓄模式

支持三种模式，在设置页中配置：

- **固定金额（fixed）：** 每月固定储蓄 RM X
- **百分比（percent）：** 储蓄月度预算的 Y%
- **两者（both）：** 同时设置固定金额 + 百分比，目标为两者之和

### 4.2 可支配预算

**可支配预算 = 月度预算 − 储蓄目标**

所有预算进度显示、每日限额计算均基于可支配预算，而非原始预算。

### 4.3 每日可支配限额

- **总额日均（未扣储蓄）：** 预算 ÷ 剩余天数
- **可用日均（已扣储蓄）：** 可支配预算 ÷ 剩余天数

两个数值同时在总览页和统计页的储蓄预测卡片中显示。

### 4.4 储蓄进度环

Canvas 圆环图，显示当前已储蓄金额占储蓄目标的百分比。状态颜色：

- **绿色：** 已达到目标（>= 100%）
- **琥珀色：** 部分完成（> 0%）
- **灰色：** 未设置或无储蓄

### 4.5 储蓄预测

基于当前日均消费推算月末储蓄，以叙事文案呈现：

- 正向预测："如果维持当前消费习惯，预计月末可存 RM X"
- 负向预测："预计超支 RM X，请注意控制支出"

预测同时显示在总览页、统计页和报告页。

## 5. 数据管理 (Data Management)

### 5.1 localStorage

所有数据持久化存储在浏览器的 `localStorage` 中，键名为 `budgetAppData`。应用初始化时自动加载，每次数据变更后自动保存。

### 5.2 JSON 导出 / 导入

- **导出：** 调用 `JSON.stringify()` 生成完整的 JSON 文件并下载
- **导入：** 文件选择器读取 JSON 文件后，用户可选择"替换当前数据"或"合并到当前数据"
  - 替换模式：完全覆盖所有数据
  - 合并模式：合并记录列表、新增不重复的分类、合并预算和储蓄设置

### 5.3 CSV 导出

生成 UTF-8 BOM 编码的 CSV 文件，列结构：

| ID | 金额 | 分类 | 日期 | 备注 | 创建时间 |
|----|------|------|------|------|----------|

备注内容中的双引号自动转义。

### 5.4 Excel 导出（XML Spreadsheet 2003）

不依赖任何 Excel 库，直接生成符合 SpreadsheetML 规范的 XML 文件。包含 5 个工作表，所有公式均为实时公式，用户可在 Excel / LibreOffice / Google Sheets 中修改数据并自动重算。

**5 个工作表：**

| 工作表 | 内容 | 公式类型 |
|--------|------|----------|
| 消费记录 | 所有记录明细（序号、日期、金额、分类、子分类、备注、月份），按日期降序 | 底部 SUM / AVERAGE / MAX / COUNT 公式 |
| 分类统计 | 每个根分类的汇总统计（序号、分类、总金额、笔数、平均、占比） | SUMIF / COUNTIF / 百分比公式 |
| 月度统计 | 每月汇总（预算、支出、日均、笔数、储蓄目标、储蓄、使用率） | SUMIF / AVERAGEIF / 自定义计算公式 |
| 预算跟踪 | 分类预算执行跟踪（分类、预算、支出、剩余、使用率、状态）带 IF 状态判断 | IF(支出>预算, "超支", "预算内") |
| 储蓄统计 | 每月储蓄明细（预算、储蓄目标、可支配、实际支出、储蓄、达成率） | 公式计算达成率 |

样式方面包含：蓝色表头、绿色表头、货币格式、百分比格式、合计行加粗灰色背景、超支红色字体等。

### 5.5 PNG 图表下载

每个 Canvas 图表下方有"下载 PNG"按钮，调用 `canvas.toDataURL('image/png')` 生成图片并触发下载。

## 6. 特色亮点 (Unique / Standout Features)

- **单文件零依赖：** 整个应用就是一个 HTML 文件，约 6,759 行，零外部库、零 CDN 引用。CSS 设计系统、JavaScript 数据层、渲染引擎、图表系统全部手写。

- **弹性动画饼图：** 饼图使用定制弹性缓出函数（非标准 ease-out），呈现弹簧 bounce 效果。Hover 时扇形弹出并带阴影和白色高亮边框，交互反馈细腻。

- **全链路下钻：** 饼图点击扇形 -> 下钻到子分类层级 -> 点击记录可编辑；柱状图点击柱条 -> 下钻子分类；热力图点击日期 -> 查看当日记录列表。统一的钻取栈管理。

- **子分类百分比预算：** 不仅支持固定金额预算，还支持百分比类型，自动按月度总预算换算实际金额。这在同类工具中不常见。

- **父子预算校验：** 保存子分类预算时自动校验子项之和不超过父级预算，防止预算超额配置。

- **实心 / 分段预算条切换：** 在预算进度卡片中一键切换单色条和按子分类分割的彩色分段条，直观展示预算分配结构。

- **双重每日限额：** 同时显示扣除储蓄前后的两种每日可支配限额，帮助用户在储蓄和消费间做出更明智的决策。

- **软删除可撤销：** 删除记录并非立即永久删除，而是 5 秒窗口期可通过 Toast 按钮撤销。如果 5 秒内再次删除同一记录，计时器重置。

- **日历热力图 vs 每日目标：** 6 色梯度基于消费与储蓄调整后的日均可支配目标的比率，而非简单的高低比较，颜色反映预算执行质量。

- **储蓄预测自然语言文案：** 系统根据当前消费趋势生成中文叙事性储蓄预测，而非仅显示数字，更友好直观。

- **深色模式图表：** 所有 Canvas 图表通过读取 CSS 变量的 `getThemeColors()` 函数自动适应深色模式，无需手动切换。

- **响应式双导航：** 桌面端显示左侧固定侧边栏，移动端显示底部标签栏，带 `safe-area-inset-bottom` 支持全面屏设备。

- **5 工作表 Excel 导出：** 生成真实 XML Spreadsheet 2003 格式，包含 SUMIF、AVERAGEIF、IF 等 60+ 处实时公式，用户可修改数据并自动重算，是同类单页应用少有的功能。

- **统一树形选择器：** 同一套分类树选择器组件复用于 7 种以上场景（记账分类选择、编辑分类选择、筛选分类选择、批量改分类、分类移动目标选择、分类合并目标选择、预算监控选择）。

- **批量操作：** 批量删除和批量修改分类，支持一次选择多条记录统一操作。

## 7. 技术架构 (Technical Architecture)

### 7.1 单文件架构

整个应用是一个独立的 HTML 文件，约 6,759 行，分为三个主要部分：

| 部分 | 行数 | 说明 |
|------|------|------|
| CSS | ~1,070 行 | 完整的组件设计系统，包含 CSS 变量主题、响应式布局、动画、打印样式 |
| HTML | ~60 行 | 静态骨架 — Toast 容器、Modal 容器、侧边栏、主内容区、底部导航 |
| JavaScript | ~5,630 行 | 数据层、渲染引擎、图表系统、路由、工具函数 |

### 7.2 数据层：DataStore 模式

- **`DataStore` 对象**：内存中的 JavaScript 对象，所有 CRUD 操作直接修改对象属性，每次修改后自动同步到 `localStorage`
- **数据模型**：`records`（记录数组）、`categories`（分类数组）、`budgets`（月度预算 Map）、`categoryBudgets`（分类预算 Map）、`savingsTarget`（储蓄目标对象）
- **记录字段**：`id`, `amount`, `categoryId`, `date`, `note`, `createdAt`, `updatedAt`
- **分类字段**：`id`, `name`, `icon`, `color`, `parentId`, `sortOrder`
- **软删除机制**：删除的记录先放入 `_pendingDelete` 缓冲区，5 秒后永久清除，期间可撤销

### 7.3 计算层：StatsEngine

- **纯计算无副作用**：所有统计计算在 `StatsEngine` 对象中实现，不修改数据状态
- **核心方法**：`getMonthTotal`, `getCategoryTotals`, `getDailyTotals`, `getDailyAverage`, `getPredictedTotal`, `getSavingsPrediction`, `getRemainingDailyLimit`, `getCategoryBreakdownDeep`, `getCustomRangeTotals`, `getOverspentCategories`, `getLast7Days`, `getMonthlyTotals`

### 7.4 Canvas 图表系统

- **原生 Canvas 2D API**：所有图表使用 `CanvasRenderingContext2D` 绘制
- **DPR 自适应**：根据 `window.devicePixelRatio` 设置物理分辨率，保持高 DPI 屏幕清晰
- **CSS 响应式**：`canvas.style.width: 100%` 配合内部尺寸计算实现自适应
- **动画引擎**：`requestAnimationFrame` 驱动的逐帧动画，支持缓出、弹性等多种缓动函数
- **深色模式感知**：`getThemeColors()` 函数读取 CSS 变量实时获取主题色
- **事件绑定**：饼图使用 `canvas._pieHandlers` 存储事件引用以便清理和重建

### 7.5 CSS 变量主题

- **设计 Token**：所有颜色、间距、阴影、圆角定义为 CSS 自定义属性（`--primary`, `--bg`, `--card-bg`, `--text-primary`, `--shadow` 等）
- **深色模式**：`[data-theme="dark"]` 选择器覆盖所有变量值
- **主题持久化**：用户偏好存储在 `localStorage`，初始化时自动加载

### 7.6 事件处理

- **全局事件委托**：部分交互通过内联 `onclick` 属性绑定，导航通过 `querySelectorAll` + `addEventListener` 统一绑定
- **导航路由**：`window.location.hash` 驱动标签切换，`hashchange` 事件监听 URL 变化，支持浏览器前进/后退
- **窗口 Resize 防抖**：200ms 防抖延迟，避免频繁重绘

### 7.7 路由系统

- **Hash 路由**：`location.hash` 控制当前页面，默认 `#overview`
- **动态渲染**：每次切换标签调用对应的 `render*()` 函数重建 DOM
- **页面切换动画**：CSS `@keyframes pageEnter` 实现 350ms 的滑入+缩放动画
