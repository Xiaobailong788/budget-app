### 构建 1 — 2026-07-09 20:43 UTC：初始发布

**概要：**
构建完成并打磨了一个完整的单页个人预算跟踪应用。整个应用包含在一个 HTML 文件（104 KB，2,895 行）中，零外部依赖。在任何现代浏览器中打开即可使用——无需安装或互联网。

**包含的功能：**
- 仪表盘（总览）：月度支出总览，包含 4 张摘要卡片、预算和储蓄进度环形图（Canvas）、7 天支出趋势迷你折线图、前 5 大支出分类、超支警告、储蓄预测及每日剩余限额
- 记账（记一笔）：带 RM 前缀的金额输入框、带树形选择器的分类选择器（表情符号 + 颜色）、日期/时间选择器、备注字段、验证及成功提示
- 流水：卡片列表，按最新时间排序，点击卡片编辑，滑动删除，高级搜索（关键词、日期范围、分类筛选、金额范围），结果计数及无限滚动
- 分类：预加载 8 个根分类及 21 个子分类，支持展开/折叠的递归树，添加根/子分类，重命名，更改表情符号图标，更改颜色，带循环防止的移动操作，带级联选项的删除
- 统计：月份选择器或自定义日期范围，摘要卡片，可交互饼图（点击下钻），带渐变填充的平滑曲线折线图，彩色编码柱状图，所有图表均配有"下载 PNG"按钮
- 设置：月度预算，储蓄目标（固定值/百分比/两者），JSON 导出/导入（替换或合并），CSV 导出，清除所有数据

**已修复的缺陷：**
- 修复了 `selectedCategoryId` 在页面间的状态泄漏（在 `renderAddPage()` 和 `closeModal()` 中添加了空值重置）
- 修复了编辑记录提交后 `selectedCategoryId` 未重置的问题
- 修复了饼图点击处理程序重复绑定导致的低效问题（添加了 `canvas._pieClickSetup` 保护标志）
- 修复了 `confirmDeleteCategory` 中的冗余代码（合并了重复分支）
- 修复了记录更新时缺失 `updatedAt` 时间戳的问题

**已应用的 UI 打磨：**
- 增强了页面切换动画，增加缩放效果
- 流畅的弹簧式提示动画
- 毛玻璃模态背景（`backdrop-filter: blur(4px)`）
- 弹簧过冲模态缩放动画
- 触感式按钮按压反馈（`::after` 伪元素）
- 卡片悬停过渡效果，含边框和阴影变化
- 改进底部导航触摸目标区域
- 增强移动端/平板/桌面端的响应式断点

**参与的智能体：**
- organizer：初始 HTML + CSS 框架
- writer：数据层、页面、图表、统计引擎
- ui-agent：视觉打磨、动画、响应式优化
- tester：语法验证、缺陷检测、回归测试
- team-leader：协调、缺陷修复、日志记录

**技术细节：**
- 单个 .html 文件（104 KB，2,895 行）
- 零依赖——无 CDN、无库、无框架
- 数据存储在浏览器的 localStorage 中
- 响应式：底部标签导航（移动端）和侧边栏（桌面端）
- 基于哈希的路由（#/overview、#/add 等）

**已知限制：**
- 用户提供的内容（备注、分类名称）通过 innerHTML 渲染，未经过滤净化（在纯本地应用中，自 XSS 风险极小）
- 仅饼图和折线图具有加载动画；柱状图无动画
- 实现了基于点击的下钻功能，但未提供悬停提示
- `updatedAt` 字段仅在更新时设置，创建时未设置
- 需要现代浏览器（ES2020+）；不支持 Internet Explorer
- 数据存储在 localStorage 中——清除浏览器数据会丢失记录（请使用 JSON 导出备份）

### 构建 2 — 2026-07-09 22:46 UTC：安全修复

**概要：**
对预算应用进行了三项安全改进。外观、功能或行为均未发生变化。

**修复 1 — 内容安全策略（CSP）：**
添加了一个 `<meta>` 标签，告知浏览器阻止所有外部资源，仅允许内联样式和脚本。这可以防止恶意脚本从外部源加载。

**修复 2 — HTML 注入防护：**
新增了一个过滤函数（`escHtml`），并将其应用于所有 29 处用户提供的文本插入页面 HTML 的位置。这可以防止攻击者通过文本字段注入恶意 HTML 标签。

受保护的字段：
- 备注（紧凑视图和卡片视图）
- 分类名称和图标（整个 UI 中——选择器、记录显示、管理对话框）
- 搜索关键词（搜索筛选输入框）
- 分类重命名/图标/颜色/移动/删除对话框

**修复 3 — 文件权限：**
文件权限收紧为 `600`（仅属主可读/写），防止系统上的其他用户读取或修改该文件。

**未变更的内容：**
- 应用外观、功能或行为
- 数据存储或导出逻辑
- Canvas 图表（它们使用安全的渲染 API）
- CSV/XML 导出函数（它们使用自身的引号处理）

**参与的智能体：**
- team-leader：任务分解、所有编辑、验证、日志记录

**值得注意的发现：**
- 文件中已包含用于 XML 导出过滤的 `esc()` 函数
- 共 29 处模板字面量表达式分布在 25 个以上位置，已全部使用 `escHtml()` 包裹

**关键设计决策：**
- 所有分类初始为折叠状态（不自动展开）
- 仅点击箭头图标进行展开/折叠（而非整个标题）
- 标题上仅暴露 2 个按钮："添加子分类"和"编辑分类"
- 所有其他操作（重命名、图标、颜色、移动、合并、删除）均置于编辑模态框中
- 子级与父级使用相同的按钮设计
- 预算输入始终显示在标题行上（折叠与展开状态均可见）
- 深度 >= 4 时显示深层嵌套面包屑导航

**已移除的内容：**
- 旧的分类树 CSS 类（`.category-tree`、`.category-node`、`.cat-toggle`、`.category-children`）
- 每个分类的 6 个独立操作按钮合并到编辑模态框
- 页面加载时根分类自动展开

**已新增的内容：**
- 新的手风琴 CSS（31 个选择器）
- 新的 `buildCategoryTreeHTML` 函数，采用手风琴结构
- `toggleCatItem`——带 CSS 动画的展开/折叠功能

- `editCategory`、`selectEditIcon`、`selectEditColor`、`saveCategoryEdit`——统一的编辑模态框

5. **预算选择模态框：** "选择要监控的分类"对话框现在显示带有预算的子分类（使用 `->` 缩进），采用新的对象格式。

7. **CSS：** 为预算进度条填充添加了过渡规则。

### 构建 5 — 2026-07-10：动画与编辑模态框修复

**修复 1 — 更流畅的手风琴动画：**

编辑模态框之前每次渲染 90 多个表情符号网格项和 14 个颜色样本，导致速度缓慢。表情符号网格和颜色选择器现在显示为简单预览，配以"更改图标"和"更改颜色"按钮，打开现有的（已轻量化的）图标/颜色选择模态框。

分类页面现在显示分类数量、手风琴使用说明以及"展开全部"/"折叠全部"链接。

### 构建 7 — 2026-07-10 01:32 UTC：饼图下钻修复

**概要：**
修复了与饼图展开视图行为和模态框尺寸相关的两个问题。

**问题 1 — 展开视图中饼图点击下钻：**
问题：在展开（模态框）视图中点击饼图切片以钻入子分类的功能不可靠。旧代码创建一个新的 canvas 并尝试在固定的 700ms 超时后包装点击处理程序，造成了脆弱的时序竞争。

修复：代码现在改为将原始饼图 canvas **移动**到模态框中。现有的点击处理程序得以保留。新增 `window._expandedPieActive` 标志，当在展开模式下点击切片时自动关闭模态框。

三项变更：
- `drawPieChart()` 点击处理程序——增加了对 `window._expandedPieActive` 的检查
- `expandPie()` 函数——重写为将原始 canvas 移动到模态框中
- `shrinkChart()` 函数——更新为将 canvas 移回并清除标志

**问题 2 — 放大模态框：**
问题：展开图表模态框限制为 560px 宽，在较大屏幕上显得局促。

修复：
- 模态框内部容器从 560px 加宽至 720px
- Canvas 最大高度从 55vh 增加到 65vh
- 热力网格和日期单元格放大以适应更大的模态框

**验证：**
- JavaScript 语法通过验证
- 所有旧的 `expandPieChart` 引用已移除
- 新的 `_expandedPieActive` 标志已正确设置、检查并清除
- 新的 `_pieOriginalParent` 变量已正确跟踪和清理

## 已知限制（汇总）

来自所有构建，记录了以下限制：

1. **HTML 过滤：** 已应用安全修复，但纯本地运行的本质意味着自 XSS 风险极小。
2. **柱状图动画：** 仅饼图和折线图具有加载动画；柱状图无动画。
3. **图表提示：** 已实现基于点击的下钻功能，但图表中未提供悬停提示。
4. **创建时的 `updatedAt`：** `updatedAt` 字段仅在更新时设置，记录创建时未设置。
5. **浏览器支持：** 需要现代浏览器（ES2020+）；不支持 Internet Explorer。
6. **localStorage 依赖：** 清除浏览器数据将丢失所有记录。请使用 JSON 导出备份。
7. **未变更的预算调用点：** 两个 `getCategoryBudget` 调用点（总览模板和月度报告表）仍将返回值视为纯数字（需要后续跟进）。
8. **模态框链式操作边界情况：** 如果用户点击"更改图标"后立即取消，编辑模态框状态已关闭（可接受，因为用户可以重新打开）。
9. **极高分类：** 基于 rAF 的手风琴未设置 3000px 回退高度，因此如果分类极高，动画可能截断（内容为静态，因此 scrollHeight 是准确的）。
